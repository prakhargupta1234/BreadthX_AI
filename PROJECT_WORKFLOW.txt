1. PROJECT OVERVIEW
-------------------
BreatheX AI is an advanced medical-grade AI platform designed to classify 
respiratory diseases (Healthy, Asthma, COPD) using audio analysis of cough sounds.
The system provides a seamless experience from audio recording to clinical-grade
prediction reports.

2. TECHNOLOGY STACK
-------------------
- Frontend: React (Vite), Framer Motion (Animations), Lucide React (Icons).
- Backend: FastAPI (Python), Uvicorn (Server).
- Database: MySQL (Relational storage for users and test history).
- Machine Learning: PyTorch, Torchaudio, Librosa.
- Authentication: JWT (JSON Web Tokens) with Argon2/Bcrypt hashing.

3. MACHINE LEARNING ARCHITECTURE
--------------------------------
- Model Core: Pretrained ResNet18 (Deep Convolutional Neural Network).
- Modifications: 
    * Input layer modified to accept 1-channel Mel-spectrograms (instead of 3-channel RGB).
    * Pretrained weights averaged across channels to maintain performance.
    * Output layer replaced with a 3-class linear classifier (Asthma, COPD, Healthy).
    * Dropout (0.5) added for regularization to prevent overfitting.
- Audio Processing:
    * All audio is resampled to 16,000 Hz for consistency.
    * Converted into Mel-spectrograms (128 mel bins, 1024 FFT window).
    * Normalized into Log-scale (dB) before being fed into the ResNet model.

4. DETAILED INTERNAL WORKING & ML PIPELINE
------------------------------------------
Step 1: Signal Acquisition & Normalization
    * The system accepts WAV audio files. 
    * Audio is loaded using the Librosa library to ensure cross-platform compatibility.
    * Signal is resampled to a fixed 16kHz sample rate to match the model's training parameters.

Step 2: Feature Engineering (Spectrogram Generation)
    * The raw time-series audio signal is converted into a Mel-Spectrogram using 
      Torchaudio's GPU-accelerated transforms.
    * Parameters: n_fft=1024, hop_length=512, n_mels=128.
    * The power spectrogram is converted to a decibel (dB) scale using `AmplitudeToDB`.
    * Result: A 2D image-like representation where Y-axis is frequency (mel-scale) 
      and X-axis is time.

Step 3: Temporal Standardization
    * To handle variable-length cough recordings, the system uses a fixed-width window 
      of 400 temporal frames (~12.8 seconds at 16kHz).
    * Short clips are Zero-Padded to reach 400 frames.
    * Long clips are Truncated at 400 frames.

Step 4: Neural Network Inference
    * The 128x400 feature matrix is fed into the modified ResNet18.
    * The first layer extracts low-level audio textures (spectro-temporal features).
    * Deeper residual blocks combine these features into high-level abstractions 
      representing respiratory patterns.
    * The final Linear layer outputs raw "logits" for each disease class.

Step 5: Probability Computation
    * A Softmax function is applied to the logits to generate a probability distribution.
    * The class with the highest probability is selected as the Primary Prediction.
    * The Confidence score is the probability of the predicted class.

5. PROJECT FLOW & SYSTEM ARCHITECTURE
-------------------------------------
1. Authentication:
    - User interacts with the React Frontend (Signup/Login).
    - Backend validates credentials against MySQL.
    - JWT Token is generated and stored for session management.

2. Prediction Pipeline:
    - User uploads a .wav file via the "Test" page.
    - Frontend sends the file to the `/predict` endpoint.
    - FastAPI invokes the `DiseasePredictor` class.
    - Inference is performed on the provided audio.
    - Results are stored in the database for the user's history.

3. Reporting & Visualization:
    - Frontend receives JSON results containing prediction and all class probabilities.
    - An interactive report modal is rendered using Recharts and Framer Motion.
    - Users can download/print the report as an official clinical document.

6. DATABASE SCHEMA
------------------
- `users`: Stores id, name, email (unique), hashed_password, and created_at.
- `predictions`: Stores id, user_id (foreign key), filename, result, confidence, 
  all_scores (JSON string), and timestamp.

7. DEVELOPMENT & DEPLOYMENT
---------------------------
- Model training was performed using Cross-Entropy Loss and Adam Optimizer.
- Data Augmentation (SpecAugment: Time & Frequency masking) was used during training 
  to increase model robustness against different recording environments.
- Deployment utilizes Uvicorn as an ASGI server for high-performance async processing.
